How to start a blog

Let me be clear from the outset. I am no webdeveloper. I have however, thoroughly enjoyed the process of learning a little bit more about the techniques and technology that go into taking a website live, and I want to share some of the learnings in this post. I’ll be linking to lots of resources that can explain all the detail.

Use Blogdown to get up and running

Blogdown is a package in R which helps you build a static website, using Rmarkdown documents as the website pages. Rmarkdown One big advantage of using Rmarkdown (.Rmd) over markdown (.md) files, is that it allows you to seamlessly include code in your blogs, very useful if you’re writing for a technical audience.

Alison hill, made a blog post using blogdown about how to make a blog post using blogdown - meta. You can find it here.

Keep things tidy

An observation i’ve had is, one of the key consideration when blogging, and when building a website in general, is keeping things tidy and therefor maintainable. Since we are making a static site, conceptually, all this boils down to is writing content and then telling the computer which content to retrieve when you click on certain things on the webpage. An organizational hurdle you will quickly run into is, how to make sure images, data and other things you want available in a given blogpost are stored in an accessible place for that blog post, this can be simplified greatly using page bundles - again, Alison Hill to the rescue.

Hugo

There are a few folders and things to know about with hugo. You can find much more information on the hugo website

Archetypes

These set up what a new post looks like when you make a post in certain folders. It sets up default front matter on posts (front matter is the stuff is the YAML header that gives meta data about the pos such as, when it was written, who it was written by and whether to include a table of contents). For example, in one folder you may want the default author to be ‘Ezra’ and in another, for the default author to be ‘Sarah’.

Shortcodes

You can do a lot with HTML and an (R)markdown (.(R)md) document accepts raw HTML. Shortcodes are the reuable snippets of HTML that save you from writting raw HTML each time you say, want to embedd a youtube video in you md. Instead just write {{< youtube w7Ft2ymGmfc >}}. I think of these as functions which output HTML - or “pre defined chucks of HTML that you can insert in your markdown file”. These are saved in layouts/shortcodes/

Shortcodes are supposed to work in plain Markdown documents only. To use shortcodes in R Markdown instead of plain Markdown, you have to call the function blogdown::shortcode(), e.g.,

{{% tweet "852205086956818432" %}}

Taxomony

Taxonomy is how you group content together. So in Hugo, by default there are 2 ways to group content together. you can use ‘tags’ or ‘categories’. Front matter (the stuff in your YAML heading) can be used to implement tags and categories. You can also create your own custom taxonomies that, when clicked, direct you to taxonomy pages, which are list pages which list out the relevant pages.

Templates

Templates are more advance, this is really just HTML templates. A hugo theme is made up of templates. In hugo there are two high level types of content, list pages and single pages. The home page is list page. The templates are for these different types of pages, the list pages and single page, look basically the same. A template is the HTML skeleton. Templates can be altered in layouts/_default they are the default templates that will be used for out file. single.html acts as a template for single pages. The {{.content}} syntax, takes the content from all of your single pages and shoves it on the website.

Hugo Variable

Thids is a value that is stored about the page and that you can use in templates. (so that not every page looks the same)

Block

baseof.html templates are an overarching teplate that you can use to stick your single.html, list.html index.html (for the home page). You stick it in there using blocks. You define blocks in single.html using

{{ define 'main' }}
This is a single page
{{ end }}

and then use it in your baseof.html template using

{{ block "main" . }}
{{ end }}

With blocks you can decide if you want to override a specific default thats on a page with a specific template.

CSS Styling

The two CSS files are placed under the static/css/ directory of the theme. In the HTML template header.html, the path /css/style.css refers to the file static/css/style.css
